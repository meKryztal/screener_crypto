# Crypto Futures Screener

Веб-скринер для фьючерсов Binance с графиком, индикаторами и детектором крупных лимитных ордеров. Работает на FastAPI + LightweightCharts, данные через WebSocket от Binance.

(все скриншоты скринера в папке screenshot)
---

## Стек

| Компонент | Технология |
|---|---|
| Backend | Python 3.11+, FastAPI, ccxt / ccxt.pro |
| Frontend | Vanilla JS, LightweightCharts 4.1 |
| Данные | Binance USDT Futures WebSocket + REST |
| Хранилище | Parquet-кэш на диске, JSON для настроек |

---

## Авторизация

Доступ к сайту защищён кодом — без него все страницы и API недоступны.

### Настройка

Создай файл `.env` рядом с `main.py`:

```bash
cp .env.example .env
```

Открой `.env` и задай один или несколько кодов через запятую:

```
ACCESS_CODES=мой-код,второй-код,третий-код
```

Сгенерировать случайный надёжный код:

```bash
python -c "import secrets; print(secrets.token_urlsafe(16))"
```

### Как работает

- При открытии сайта без авторизации — автоматический редирект на `/login`
- После ввода правильного кода браузер получает куку, которая живёт **30 дней**
- Повторно вводить код не нужно пока кука не истекла или не был отозван доступ
- Выход: открыть `/logout`

### Управление доступом

Все изменения применяются **без перезапуска сервера** — достаточно сохранить `.env`.

| Действие | Что сделать |
|---|---|
| Добавить пользователя | Добавить новый код в `ACCESS_CODES` |
| Отозвать доступ одному | Удалить его код из `ACCESS_CODES` — кука сразу перестаёт работать |
| Сбросить всех | Заменить все коды на новые |

### Защита от брутфорса

После **5 неверных попыток** IP блокируется на **5 минут**. Счётчик сбрасывается автоматически.

### Важно

Добавь `.env` в `.gitignore` чтобы коды не попали в репозиторий:

```bash
echo ".env" >> .gitignore
```

## Архитектура данных

### Двухуровневое хранилище

| Уровень | Что хранит | Для чего |
|---|---|---|
| `mini_store` | Последние 2000 баров 1m для всех символов | Скринер (vol, spike, pump) |
| `chart_store` | 50 400 баров 1m (35 дней) для активных символов | График, индикаторы |

`chart_store` выгружается из RAM через 10 минут неактивности. При повторном открытии — догружает только недостающие бары из кэша на диске.

### Кэш на диске

- Формат: Parquet (`.cache/`)
- Срок хранения: **35 дней** (старые строки обрезаются при загрузке)
- Инкрементальное обновление: при старте догружает только бары с последнего кэшированного timestamp

## Скринер

Список всех USDT-фьючерсов Binance с сортировкой в реальном времени.

### Вкладки сортировки

| Вкладка | Описание |
|---|---|
| **Vol 24h** | Объём в $ за последние 24 часа |
| **2H/24H** | Объём за 2 часа / средний 2-часовой за сутки |
| **Pump** | Рост от минимума за последние 3 часа (%) |
| **Big Bid/Ask** | Ближайший аномально крупный лимитный ордер |

### Big Bid/Ask (детектор лимиток)

- Каждые 30 сек запрашивает стакан (100 уровней) для всех символов
- Хранит снапшоты за последние 5 минут
- Уровень считается аномальным если `qty / median(all_qtys) >= mult` (по умолчанию 8x)
- **Антиспуффинг**: уровень должен присутствовать во всех снапшотах старше 60 секунд
- В скринере показывает `↑0.123%` (красный = ask/сопротивление) или `↓0.045%` (зелёный = bid/поддержка)

## График

### Таймфреймы

`1m` · `5m` · `15m` · `1h`

HTF свечи (5m, 15m, 1h) строятся ресемплингом из 1m данных в реальном времени.

### Обновление свечей

Новые закрытые свечи добавляются автоматически: таймер срабатывает через 2 секунды после закрытия текущей свечи по серверному времени. Незакрытая свеча не отображается.

## Индикаторы

### Cluster (поглощение объёма)

Порт Pine Script индикатора absorption на Python.

- Находит свечи где mid-price попадает в тень (выше тела для бычьих, ниже для медвежьих)
- Объём свечи должен превышать N-й перцентиль за lookback баров (Auto режим) или заданный порог (Manual)
- На HTF: показывает **все** 1m бары с поглощением внутри HTF свечи, а не только один лучший
- Размер пузыря: 1x / 2x / 3x / 4x от порога

### POC (Point of Control)

Линии уровней максимального объёма за период.

| POC | Цвет | По умолчанию |
|---|---|---|
| 4H | Синий `#1f77b4` | Вкл |
| 1D | Оранжевый `#fc7601` | Вкл |
| 1W | Зелёный `#298b29` | Вкл |
| 1M | Фиолетовый `#9467bd` | Выкл |

### SMA 20 / EMA 200

Скользящие средние по цене закрытия.

### Volume / Delta / OI

Отдельные подграфики под основным.

- **Volume** — объём свечей, цвет по направлению
- **Delta** — разница buy/sell volume (оценка через close/high/low)
- **OI** — открытый интерес с Binance `/futures/data/openInterestHist`, обновляется с периодом TF

### DOM (стакан)

Визуализация книги ордеров прямо на графике справа от цены.

- Глубина подстраивается под видимый ценовой диапазон автоматически
- Нормализация баров по 95-му перцентилю видимых уровней (большая одиночная лимитка не сжимает остальные)
- Спред между лучшими bid/ask отмечается пунктирной линией

### Big Bid/Ask (на графике)

Горизонтальные линии аномальных лимитных ордеров прямо на ценовом графике.

- Линия начинается от момента **первого обнаружения** уровня
- Зелёная = bid (поддержка), красная = ask (сопротивление)
- Подпись: `5m $2.4M 9.2x` — сколько живёт / размер в долларах / во сколько раз больше нормы
- Мультипликатор аномалии настраивается в панели Indicators (`mult`)

## Инструменты рисования

| Инструмент | Описание |
|---|---|
| Горизонтальная линия | Одиночная линия по цене |
| Линия тренда | Две точки, экстраполяция за пределы данных |
| Прямоугольник | Зона на графике |
| Фибоначчи | Уровни 0 / 23.6 / 38.2 / 50 / 61.8 / 78.6 / 100% |
| Volume Profile | Профиль объёма с POC, VAH, VAL (70% зона) |
| Позиция Лонг/Шорт | Визуализация сделки с TP/SL и R:R |
| Карандаш | Свободный рисунок |

Все рисования сохраняются на сервере (`drawings.json`), срок хранения 35 дней.

## Настройки

Все пользовательские настройки сохраняются на сервере в `settings.json`:

- Активный символ и TF
- Состояние POC (вкл/выкл по каждому периоду)
- Состояние индикаторов
- Тема (тёмная / светлая)
- Видимость скринера
- Режим и порог кластеров

## Конфигурация (main.py / data_manager.py)

```python
# data_manager.py
MINI_BARS        = 2000       # баров в mini_store (скринер)
CHART_LIMIT      = 50_400     # баров в chart_store (35 дней × 1440)
CHART_EVICT_SEC  = 1200        # сек до выгрузки неактивного символа из RAM
CACHE_WEEK_SEC   = 35*24*3600 # срок хранения кэша на диске

# main.py
MAX_DISPLAY = {"1m": 6000, "5m": 6000, "15m": 6000}  # баров на фронт

# DOM аномалии
_DOM_ANOMALY_MULT  = 8.0  # минимальный множитель аномалии
_DOM_MIN_PERSIST   = 60   # секунд — минимальное время жизни уровня
_DOM_DEPTH         = 100  # уровней стакана
_DOM_POLL_INTERVAL = 30   # секунд между циклами поллера
```

## API

| Endpoint | Описание |
|---|---|
| `GET /login` | Страница входа |
| `GET /logout` | Выход (сбрасывает куку) |
| `GET /api/symbols` | Список всех символов со скринерными метриками |
| `GET /api/bars/{symbol}?tf=5m` | OHLCV данные |
| `GET /api/absorption/{symbol}` | Кластеры поглощения |
| `GET /api/poc/{symbol}` | POC сегменты |
| `GET /api/dom/{symbol}?depth=100` | Стакан |
| `GET /api/oi/{symbol}?tf=5m` | Открытый интерес |
| `GET /api/limit_levels/{symbol}` | Аномальные лимитки для графика |
| `GET /api/anomalies` | Аномалии по всем символам (для скринера) |
| `POST /api/set_limit_mult?mult=8` | Изменить порог аномалии на лету |
| `GET /api/settings` | Загрузить настройки |
| `POST /api/settings` | Сохранить настройки |
| `GET /api/drawings` | Загрузить рисования |
| `POST /api/drawings` | Сохранить рисования |
| `GET /api/dom_debug/{symbol}` | Отладка снапшотов стакана |

> Все `/api/` endpoints требуют авторизации. Без куки возвращают `401 Unauthorized`.
